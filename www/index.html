Project Info

  • Project Name: AppFuse
  • Latest Release: 1.7
  • Latest Release Status: Production
  • Project Type: Starter Web Application
AppFuse is an application for "kickstarting" webapp development. Download, extract and execute ant new -Dapp.name=yourApp -Ddb.name=database to instantly be up and running with a Tomcat/MySQL app. Uses Ant, XDoclet, Spring, Hibernate (or iBATIS), JUnit, jMock, StrutsTestCase, Canoo's WebTest, Struts Menu, Display Tag Library, OSCache, JSTL and Struts (Spring MVC, WebWork, Tapestry and JSF are also options). To learn more about AppFuse, its history, goals and future, checkout AppFuse: Start Your J2EE Web Apps on java.net.

The Spring Framework has greatly enhanced AppFuse since February 2004. It's used throughout for its Hibernate/iBATIS support, declarative transactions, dependency binding and layer decoupling. This clean and simple framework has greatly reduced the complexity of AppFuse, and also eliminated many lines of code. In short, for J2EE - it's the best thing since sliced bread.

Download and Installation

To download and install AppFuse, follow these simple steps:
  1. Download the source version or checkout the appfuse module from CVS (:pserver:guest@cvs.dev.java.net:/cvs).
  2. Install J2SE 1.4+, set a JAVA_HOME environment variable, install Ant 1.6.2+, set an ANT_HOME environment variable.
  3. Install MySQL 3.23.x+ (recommend 4.0.21) and Tomcat 4.1.x+ (recommend 5.0.28). Set a CATALINA_HOME environment variable to point to your Tomcat installation. Optionally, you can see my development environment setup and get links to download the above packages.
  4. Setup a local SMTP server or change mail.properties (in the web/WEB-INF/classes directory) and build.properties (in the root -- for log4j messages) to point to an existing one - they default to localhost.
  5. Copy lib/junit3.8.1/junit.jar to $ANT_HOME/lib.
  6. Run ant new -Dapp.name=YOURAPPNAME -Ddb.name=YOURDBNAME. This will create a directory named YOURAPPNAME in the same directory as appfuse.
  7. Navigate to the new directory and run ant setup to create the database and setup your app on Tomcat. The database setup will only work if your root user has no password. You can change this in build.properties if necessary. If you want to test and make sure everything works, run ant test-all - make sure Tomcat is stopped when you do this.
  8. Run ant test-reports - there will be a message after it runs telling you how you can view the generated reports.
  9. See the QuickStart Guide for optional installs (i.e. iBATIS, Spring MVC and WebWork) and how to change your package names.

Features

AppFuse has the following features:
  • Cross-platform, implemented using JavaTM technology.
  • Should run on all J2EE 1.3+ compatible application servers that support J2SE 1.4.
  • Container Managed Authentication, Remember Me, Self Registration, Password Hint, GZip Compression
  • Support for SSL throughout, including login only
  • i18n, with built-in translations for Dutch, Brazilian Portuguese and Chinese
  • Label Tag Library for displaying validation errors next to fields
  • Rich Unit Testing environment using DBUnit, JUnit, jMock and Canoo's WebTest
  • Easy configuration and rapid development using XDoclet and Spring
  • Web Framework Options: Struts, Spring MVC, WebWork, JSF and Tapestry

Build Instructions

To build AppFuse, follow these steps:
  1. Download and install ant 1.6.2 or greater from http://ant.apache.org/, or use the command asant provided with the J2EE 1.4 SDK.
  2. Download and install MySQL 3.23.53 or greater from http://www.mysql.com/downloads/mysql-4.0.html. By default, AppFuse expects a "root" user with no password (mysql defaults to this).
  3. Download and install Tomcat 4.1.x or greater from http://jakarta.apache.org/site/binindex.cgi. As part of the installation, make sure an admin/admin user exists with a "manager" role in $CATALINA_HOME/conf/tomcat-users.xml.
  4. Check out the appfuse source code from java.net. You can find instructions for doing so here: http://appfuse.dev.java.net/source/browse/appfuse/
  5. Run 'ant setup'. An appfuse database will be created in MySQL and a WAR will be expananded into $CATALINA_HOME/webapps.
  6. Login to the application at http://localhost:8080/appfuse.

Support

AppFuse is a community-supported open-source product. You can find support by subscribing to the various mailing lists offered in this java.net project. You can also use the Discussion Forums or checkout the AppFuse FAQ.

Participation

At this point, the best way to participate in this project is to use AppFuse. By using it, you will likely find some things that you'd like to improve. Talk to us on the mailing list and let us know what you'd like to see improved.

Goals for the next release include improved useability with IDEs and better appserver support (reducing coupling with Tomcat).
